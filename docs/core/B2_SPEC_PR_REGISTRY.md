## Spec-PR registry

> Assembled from the `spec-blocked` and `provisional-vector` step-4 flags (the flags that cannot become conformance vectors until a spec decision lands). Every flag was triaged against the cited authority docs (DA agreement, DECISIONS, WIRE_FORMAT, AUCTION, ACQUISITION_STATE_MACHINE, ISSUANCE_FEE, RECOVERY_INVOKE_SPEC, LAUNCH, OPEN_QUESTIONS) and the B2_KERNEL_HARDENING cross-area conflict ledger (C1–C22). Many flags across areas point at one underlying gap; they are merged below. The grouping rule: **Part A** = the genuinely missing-spec PRs (no spec text legally chooses a verdict; deriving one would invent a rule); **Part B** = the `provisional-vector` flags, grouped under the live provisional decision (#49 da-windows, #50 recovery-auth) they ride — these are not new spec PRs, they author as vectors with `decisionDeps`+`flipMarker` the moment DK ratifies (or flip if DK reverses).

Priority legend: **P0** = blocks B2 vector authoring now (a kernel predicate cannot be written without it); **P1** = needed before the dependent vectors lock but not blocking the first B2 pass; **P2** = later/launch-freeze or B3-gated, vectors wait but B2 kernel interface can stub it.
