## Decisions

- Keep `RECOVER_OWNER` compact on chain and treat variable-size BIP322 wallet
  signatures as off-chain proof envelopes.
- Use a fixed recovery proof commitment shape for requests: 32-byte wallet-proof
  hash plus a reserved zero half, fitting the existing fixed-size field.
- Keep current owner-key cancellation as a direct owner-key Schnorr signature.
- Do not make the indexer silently depend on hidden resolver-side proof state
  until proof distribution and event authoring are explicit.
- Require verified wallet-proof availability before accepting a `RECOVER_OWNER`
  request into `recovery_pending`. An on-chain request without the matching
  proof is ignored; if it also spends the live immature bond, normal bond-break
  rules can release or invalidate the name.
